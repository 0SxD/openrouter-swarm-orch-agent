# Rule 04 — No Public Actions Without Approval

## The Rule

No public-facing action without Sage's explicit approval in the current conversation:

1. `git push` (to any remote)
2. GitHub: create/edit/close issues, discussions, PRs, repo settings
3. Posting to any external platform
4. Publishing or releasing anything
5. Sharing project information externally
6. Any action visible outside this workspace

## Protocol

1. STOP before the action
2. Describe exactly what you're about to do
3. Wait for explicit "yes" or approval
4. Execute only after approval
5. Even if Sage approved a similar action earlier — ask again for each new one
